# Cloud Infrastructure Operations Lab

Production-style portfolio lab demonstrating junior Cloud Operations Engineer and Infrastructure Engineer I responsibilities across AWS and Azure.

## Business Scenario
Northstar Retail operates customer-facing workloads in AWS and internal business services in Azure. This repository documents a simulated operations environment supporting availability, security, patching, monitoring, backup, incident response, change control, cost management, and automation.

## What This Repository Proves
- AWS and Azure infrastructure operations
- Terraform infrastructure as code
- Linux and Windows administration
- Monitoring, alerting, and on-call response
- IAM, RBAC, secrets, and security baselines
- Backup validation and disaster recovery
- Incident, problem, and change management
- Cloud cost and capacity analysis
- PowerShell, Bash, and Python automation

## Portfolio Metrics
- 100 resolved cloud operations tickets
- 30 runbooks
- 20 SOPs
- 35 knowledge-base articles
- 16 operational datasets
- AWS and Azure Terraform modules
- 20+ automation scripts
- Incident reports, RCAs, change records, and executive reports

## Architecture
AWS hosts the public web platform using VPC, ALB, EC2 Auto Scaling, RDS, S3, CloudWatch, SNS, and IAM. Azure hosts internal services using VNet, virtual machines, Log Analytics, Azure Monitor, Recovery Services Vault, Key Vault, and RBAC.

All company names, users, systems, incidents, and datasets are synthetic and created for portfolio demonstration.
