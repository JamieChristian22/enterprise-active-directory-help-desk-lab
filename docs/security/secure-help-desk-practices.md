# Secure Help Desk Practices

- Use named administrative accounts; never share credentials.
- Apply least privilege and delegated administration.
- Verify identity before credential, MFA, or recovery actions.
- Never ask a user to send a password by email or chat.
- Do not disable endpoint protection to “test” an issue without approval.
- Record commands and configuration changes in the ticket.
- Use approved remote-support tools and obtain user consent.
- Lock the support workstation when unattended.
- Treat unexpected MFA prompts, repeated lockouts, impossible travel, and new forwarding rules as security indicators.
- Escalate suspected compromise rather than repeatedly resetting passwords.
